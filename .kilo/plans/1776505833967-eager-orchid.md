# CobraPix - Análise de Funcionalidades Implementadas

## 1. Core: Faturamento, Modelagem de Dados e Arquitetura Multi-tenant

### Modelagem de Dados (Prisma)

**Entidades principais implementadas:**
- `Company` — Entidade tenant com dados corporativos, ID da instância WhatsApp e status de conexão
- `User` — Funcionários da empresa (pertence à Company)
- `Debtor` — Clientes devedores (único por company + phone)
- `Invoice` — Registros de cobrança com amount, dueDate, status, pixPayload
- `CollectionLog` — Trilha de auditoria para todas as mensagens WhatsApp enviadas
- `Plan` — Planos de assinatura
- `Subscription` — Status da assinatura (TRIAL, ACTIVE, PAST_DUE)

**Arquitetura Multi-tenant:**
- Toda entidade possui foreign key `companyId`
- Queries filtram por `companyId` via token JWT
- Unique constraints em Company (document, email)

### Lógicas de Faturamento

- **Cron Jobs automáticos**: Disparo às 9:00 AM e 5:00 PM diariamente
- **Processo de cobrança**: Busca invoices overdue → deduplica envios do dia → envia WhatsApp → loga em CollectionLog
- **Importação CSV**: Endpoint `/invoices/import` aceita até 5.000 linhas, faz upsert de debtors
- **Execução manual**: Endpoint `/billing/run` para trigger manual

## 2. Integrações: Evolution API e Gateways de Pagamento

### Evolution API (WhatsApp)

**Status: Implementado**
- Container Docker (porta 8080) usando imagem `atendai/evolution-api`
- Criação de instância por empresa (nome: `cobrapix_{companyId}`)
- Geração de QR Code para pareamento
- Monitoramento de estado de conexão
- Envio de mensagens de texto
- Webhook para atualizações de conexão

**Endpoints backend:**
- `POST /whatsapp/instance` — Criar + conectar nova instância
- `GET /whatsapp/status` — Verificar status da conexão
- `POST /whatsapp/disconnect` — Desconectar instância

### Gateways de Pagamento

**Status: ✅ Implementado**
- Integração com Asaas para PIX e Boleto
- Campos `gatewayToken`, `pixPayload`, `pixExpiresAt` no schema Invoice
- Endpoints: `/payments/create`, `/payments/boleto`, etc.
- Webhook `/webhooks/asaas` para atualização automática de status

## 4. Infra e Segurança: Autenticação e Proteção de Rotas

### Autenticação

**Backend (NestJS + Passport):**
- JWT Strategy com Passport.js
- Token expira em 7 dias
- Credenciais: email + password (bcrypt hashado)
- Endpoints: `/auth/login`, `/auth/logout`, `/auth/session`

**Frontend (NextAuth v5):**
- Provider: Credentials
- Fluxo: login → chama backend → armazena JWT em cookie
- Todas as requisições incluem Bearer token no header

### Proteção de Rotas

- Middleware de autenticação no backend
- Guards JWT para rotas protegidas
- Decorator customizado `@GetUser()` para injetar companyId

### Configurações de Ambiente

```
Backend: PORT=3001, DATABASE_URL, EVOLUTION_API_URL, EVOLUTION_API_KEY, JWT_SECRET
Frontend: NEXT_PUBLIC_API_URL, NEXT_PUBLIC_APP_URL
```
